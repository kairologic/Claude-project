  - Returns: { error: { code, message } }

Validation:
  - Practice ID matches authenticated user's access
  - Date range format validation
  - Filter value validation
  - Date logic (start < end)

Location: /chrome-extension/content/content.ts (427 lines)
-----------------------------------------------------------
Purpose: Content script injected into healthcare portals
Functions:
  - detectNPI(): Main NPI detection dispatcher
  - detectNpiFromNPPES(): Query param extraction
  - detectNpiFromPECOS(): Form field + regex search
  - detectNpiFromCAQH(): Data attribute + regex search
  - detectNpiFromPayerPortal(): Generic selector + regex search
  - injectSidebar(corrections): Create and inject sidebar
  - createCorrectionCard(correction, index): Build card element
  - markCorrectionAsComplete(workflowId): API call to mark done
  - escapeHtml(text): XSS prevention

NPI Detection Algorithm:
  NPPES: ?npi=<npi> query param
  PECOS: input[name*="npi"] or /NPI[:\s]+(\d{10})/i regex
  CAQH: [data-npi] attr or /(?:NPI|#)[:\s]*(\d{10})/i regex
  Payers: Generic selectors + /(?:NPI|Provider #)[:\s]*(\d{10})/i regex

Sidebar Features:
  - Fixed position (right edge, 320px width)
  - Navy header with correction count
  - Close button (×)
  - Scrollable body
  - Per-correction card layout:
    * Field label (bold)
    * System name (small text)
    * Incorrect value (red, marked)
    * Correct value (green, marked)
    * Two action buttons: Copy, Mark Done
  - Styled with inline CSS
  - Design matches KairoLogic brand

API Integration:
  - Fetches from: /api/extensions/corrections?npi=<npi>
  - Headers: Authorization: Bearer <token>
  - Cache-friendly (could add 5-min cache)

Location: /chrome-extension/popup/popup.ts (277 lines)
-------------------------------------------------------
Purpose: Popup UI logic and authentication
Functions:
  - authenticateUser(email, password, apiBaseUrl): Login flow
  - logoutUser(): Clear auth and update UI
  - showStatus(message, type): Display status message
  - showError/showSuccess/showInfo(message): Convenience functions
  - showLoginView/showAuthenticatedView(): UI switching
  - updateUIState(): Refresh UI based on auth state
  - updateCurrentNPI(): Get NPI from content script
  - updateLastUpdated(): Update timestamp

Authentication Flow:
  1. User enters email/password/API URL
  2. POST to <apiBaseUrl>/api/auth/callback with credentials
  3. Receive JWT access token
  4. Store in chrome.storage.sync:
     - authToken
     - apiBaseUrl
     - userEmail
     - authenticated: true
     - authenticatedAt: ISO timestamp
  5. Switch to authenticated view

Event Listeners:
  - Login button: trigger authentication
  - Logout button: clear storage, switch to login
  - Enter key in email: focus password
  - Enter key in password: submit form
  - Tab switch: update current NPI display
  - Storage changes: reflect auth state

Location: /chrome-extension/background/service-worker.ts (247 lines)
---------------------------------------------------------------------
Purpose: Background service worker for token refresh and badge updates
Functions:
  - refreshAuthToken(): Validate and refresh JWT token
  - scheduleTokenRefresh(): Setup 1-hour interval refresh
  - updateBadgeCounter(): Calculate and display badge count
  - scheduleBadgeUpdates(): Setup 5-minute interval updates
  - Message handlers: getAuthStatus, refreshBadge, logout

Token Refresh:
  - Every 1 hour (TOKEN_REFRESH_INTERVAL)
  - Validates token by calling /api/extensions/corrections?npi=0000000000
  - On 401: clears auth, user must re-login
  - Logs results to console

Badge Updates:
  - Every 5 minutes (BADGE_UPDATE_INTERVAL)
  - On tab change
  - On page load complete
  - Fetches pending count for current NPI
  - Shows red badge with count (max 99)
  - Updates: chrome.action.setBadgeText()
  - Stores: lastBadgeCount, lastBadgeUpdate

Event Listeners:
  - chrome.tabs.onActivated: Update badge on tab switch
  - chrome.tabs.onUpdated: Update badge on page load
  - chrome.storage.onChanged: Clear badge on logout
  - chrome.runtime.onMessage: Handle async messages

Message Protocol:
  getAuthStatus:
    Request: { action: 'getAuthStatus' }
    Response: { authenticated: boolean, email: string | null }
  
  refreshBadge:
    Request: { action: 'refreshBadge' }
    Response: { success: boolean }
  
  logout:
    Request: { action: 'logout' }
    Response: { success: boolean }

Storage Used:
  chrome.storage.sync:
    - authToken: string
    - apiBaseUrl: string
    - authenticated: boolean
  
  chrome.storage.local:
    - lastBadgeCount: number
    - lastBadgeUpdate: string (ISO)
