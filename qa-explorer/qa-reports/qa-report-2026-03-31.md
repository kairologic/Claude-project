## Critical Findings

### 1. Blog post excerpts render raw markdown instead of HTML

- **Section:** Public Site — Blog (`/blog`)
- **Element:** Blog post card excerpts
- **Description:** At least 2 blog posts display raw `##` markdown heading syntax in the excerpt/preview text instead of rendered HTML. The first post ("Why Real-Time NPPES Delta Monitoring Should Be Your Practice's New Standard") shows `## The $62,000 Problem Hiding in Your Provider Files` as raw text. This is user-facing on the public marketing site and looks unprofessional.
- **Fix:** The blog rendering pipeline needs to either strip markdown syntax from excerpts or render it as HTML before display.

### 2. Admin Overview KPI counts don't match section pages (data inconsistency)

- **Section:** Admin Overview vs Alerts & Issues pages
- **Element:** KPI cards on `/admin/dashboard` vs `/admin/alerts` and `/admin/issues`
- **Description:** The Admin Overview shows **7 Active Alerts** and **12 Open Issues**, but the Alerts page (`/admin/alerts`) shows **0** across all severity categories ("No alerts found for the selected filter"), and the Issues page (`/admin/issues`) also shows **0** across all statuses ("No feedback yet"). Either the Overview KPI cards are displaying stale/hardcoded data, or the Alerts and Issues pages are failing to load their data from Supabase.
- **Fix:** Investigate whether Overview KPIs are hardcoded mock data vs live queries. If live, check that the Alerts/Issues pages query the same underlying tables.

### 3. User-facing dashboard inaccessible (auth redirect loop)

- **Section:** Dashboard Login
- **Element:** `/dashboard` route
- **Description:** Navigating to `/dashboard` always redirects to `/dashboard/login`, even with an active admin session. The preview URL (`/preview/demo-north-texas-med`) also redirects to login. The QA explorer script's login journey assumes email/password auth at `/login`, but the actual login page is at `/dashboard/login` and uses magic link (email-only, no password field). This means the automated QA script cannot test the user-facing dashboard at all in its current form.
- **Fix:** (a) Update QA explorer journey to use `/dashboard/login` and handle magic link flow, or add a test account with password auth. (b) Investigate why admin session doesn't grant dashboard access — these may be separate auth contexts (admin access code vs Supabase auth).

## Warnings

### 4. Practice list URL displays full UTM-tagged URL (visual clutter)

- **Section:** Practices (`/admin/practices`)
- **Element:** First practice row — website URL field
- **Description:** "BRUSHY CREEK FAMILY PHYSICIANS PA" shows the full URL including UTM parameters: `brushycreekfamilyphysicians.com/?utm_campaign=corp_listings_mgmt&utm_source=google&utm_medium=business_listing`. This makes the row visually cluttered and harder to scan. Other practices show clean URLs.
- **Fix:** Strip UTM parameters when displaying website URLs in the practice list, or truncate with ellipsis.

### 5. Inconsistent practice count: Overview says 24, Practices page shows 3

- **Section:** Admin Overview vs Practices
- **Element:** "Total Practices" KPI card vs actual practice list
- **Description:** The Overview KPI card shows **24 Total Practices**, but the Practices Management page only lists **3** practices (Brushy Creek Family Physicians PA, Brushy Creek Family Providers, Denton Family Medicine). This reinforces the finding that Overview KPIs may be mock/stale data.

### 6. Content Studio right panel is empty until post selected

- **Section:** Content Studio (`/admin/content-studio`)
- **Element:** Right preview panel
- **Description:** The right side of the Content Studio shows "Select a post from the queue to preview" as a blank placeholder taking up ~50% of the viewport. While functional, this is wasted space on initial load. Consider showing the most recent post preview by default.

### 7. Brushy Creek Family Providers shows "Pending" scan status with 0/0 payer listed

- **Section:** Practices (`/admin/practices`)
- **Element:** Second practice row
- **Description:** "Brushy Creek Family Providers" shows Last Scan status as "Pending" (3d ago) and Payer Listed as "0/0". If the scan was 3 days ago, it should have either completed or errored — "Pending" for 3 days suggests a stuck scan job. Also 0/0 payer listed with 5 providers may indicate the payer sync hasn't been run.

### 8. QA Explorer script needs journey updates for actual site structure

- **Section:** QA Explorer tool (`qa-explorer/qa-explorer.mjs`)
- **Element:** JOURNEY configuration
- **Description:** The script's journey assumes:
  - Login at `/login` (actual: `/dashboard/login`)
  - Email + password fields (actual: magic link, email only)
  - Dashboard routes at `/dashboard/workflows`, `/dashboard/providers`, etc.
  - But the admin console is at `/admin/*` with separate auth

  The script needs a parallel admin journey or updated routes to be useful.
